关于 RDK 文档矩阵

RDK 文档中心（Portal） 是 D-Robotics 开发者文档的总入口，本身不承载具体的产品/OS/算法内容，而是作为导航门户聚合所有子站。

1. 为什么拆成多个站点？​

原来所有内容聚合在同一个 Docusaurus 里，带来了几个问题：

  • 发版节奏不同：OS 版本（V3.0.0 / V3.5.0 / V4.0.5 ...）要和产品文档解耦，才能独立打版本。
  • 权责不清：产品、OS、TROS、工具链团队都要往同一个仓库改，评审冲突和打包时间都在放大。
  • 索引/搜索变慢：单仓内容过多，本地搜索索引体积和冷启动都变差。

拆分后：

关注点效果
发版节奏每个子站独立发版、独立 Release Note
CI/CD每个子站独立构建、独立部署、互不阻塞
文档版本化各子站可使用 Docusaurus versioning 管理多版本
跨站导航Portal 首页卡片 + 各子站 navbar 的「返回文档中心」

3. 部署建议​

两种可选方案：

方案 A：单域名 + 反向代理​

developer.d-robotics.cc/            → portal
developer.d-robotics.cc/product/*   → 各 product-* 子站
developer.d-robotics.cc/os/*        → 各 os-* 子站
developer.d-robotics.cc/tros/       → tros 子站
...

Nginx 层配置 location /product/rdk-x3/ { proxy_pass ...; } 即可，子站 docusaurus.config.js 的 baseUrl 保持与路径一致。

方案 B：独立子域名​

docs.d-robotics.cc/                → portal
rdk-x3.docs.d-robotics.cc/         → product-rdk-x3
rdk-x3-os.docs.d-robotics.cc/      → os-rdk-x3
...

此时每个子站可以 baseUrl: "/"，在 portal/src/data/sites.js 里把 href 改成完整 URL。

4. 版本切换​

已开启多版本的子站（例如 OS 系列）在各自 navbar 右侧有 版本下拉，可在 latest / V3.5.0 / V3.0.0 之间切换。版本通过 Docusaurus 官方的 versioned_docs/ + versions.json 管理：

  • 新版本发布：在子站目录下执行 npm run docusaurus docs:version V4.1.0（或用我们的脚手架重新生成）
  • 老版本修正：直接编辑 versioned_docs/version-X.Y.Z/ 下的对应文件
